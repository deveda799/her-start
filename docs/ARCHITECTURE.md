# 她来开局架构

## 页面与数据流

根页面在客户端完成四问、校验、本地存储与结构化结果渲染。浏览器只调用 `POST /api/analyze`；Route Handler 校验输入、限流、读取服务端模型环境变量、调用 OpenAI 兼容接口并再次执行 Zod 校验。未配置模型、超时或异常时返回明确标注的演示案例。

## 安全边界

密钥与模型地址只存在于服务端环境变量。日志不记录回答和模型原文，只记录匿名请求标识、成功状态、耗时和调用次数。前端不注入 HTML，只按已校验字段渲染。

## MVP 限制

IP 小时与项目每日限流使用单进程内存，重启会丢失且多实例不共享，仅适合比赛 MVP。AI 分析、额度判断和 localStorage 积分彼此分离。正式运营应替换为共享 Redis/云限流服务。

## 演进

免费人生资产初诊 → 付费完整商业化报告 → 人工解读/工作坊 → 副业陪跑或女性商业圈会员。账号、订单与历史报告仅预留方向，不在本次范围。
