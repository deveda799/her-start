# 中国大陆部署准备

## 比赛快速方案

使用已有腾讯云轻量服务器或支持 Node.js 20、服务端环境变量和 HTTPS 的大陆可访问环境。执行 `npm ci && npm run build`，以 `npm run start -- -H 127.0.0.1 -p 3000` 运行，Nginx 反向代理并配置有效 HTTPS 证书。部署平台中配置 `.env.production.example` 的变量；`NEXT_PUBLIC_SITE_URL` 填写正式 HTTPS 访问地址，用于分享图片绝对地址解析。`AI_API_KEY` 可以留空，留空时首页仍可完整进入明确标注的演示模式。密钥不要写入仓库或聊天。

如果先用已有腾讯云服务器做 HTTP 连通性检查，可在服务器上运行：

```bash
sudo bash scripts/deploy-tencent-opencloudos.sh
```

脚本默认使用 `http://175.178.174.40` 做公网 health 检查。注意：公网 IP 或 HTTP 只能说明大陆网络能连通服务器，不能替代比赛要求的 HTTPS 验收。最终提交给评委前必须绑定域名并配置证书，或换用已验证大陆可访问的 HTTPS 部署平台。

## 长期方案

使用腾讯云/阿里云大陆地域服务器、正式域名、进程管理、日志轮转和 Redis 限流。大陆境内服务器绑定正式域名通常需要 ICP 备案；未备案临时域名不可作为长期方案。发布采用版本目录或镜像标签，保留上一版本并以健康检查确认后切流，异常时回滚。

## 验收

通过移动数据、Wi-Fi 与微信内置浏览器打开 HTTPS 地址，完成四问、AI/演示分析、三卡浏览与重开流程；检查中文输入、键盘、安全区、返回恢复及无跨境公共 CDN 请求。AI 服务商只需保持 OpenAI 兼容协议，替换 `AI_BASE_URL`、`AI_MODEL` 和 `AI_API_KEY` 后重启服务，无需修改前端。
