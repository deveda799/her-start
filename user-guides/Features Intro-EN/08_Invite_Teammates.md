# Invite Teammates to Collaborate

## Organization Overview

- A default organization is automatically created when you register; roles are limited to **Owner** (creator) and **Member**
- Every project belongs to an organization; projects from different organizations are **invisible to each other**
- All Agent usage costs follow the **project Owner** (including both Team Agent and Private Agent)

## How to Open Member Management

Click the **"person+" icon** in the project's right sidebar (the number next to the icon shows the current member count) to open the member management panel.

The panel has three tabs at the top:

| Tab | Description |
|-----|-------------|
| **Members** | View all current project members — shows name, avatar, and email; Owner has a label |
| **Invite** | Invite new members to join the project |
| **Add Agent** | Add a previously created AI Agent as a project member for collaboration |

## Invitation Methods

| Method | Steps | Result |
|--------|-------|--------|
| **Add from organization** | Search Organization Contacts in the Invite tab, select, and click **Invite** | Organization member joins the current project |
| **Invite external members** | Click **"Add external members"** at the bottom of the Invite tab | External member **joins only the current project, not the organization** |
| **Add AI Agent** | Select a created Agent in the Add Agent tab | AI partner joins the project for collaboration |

> Inviting external members is perfect for cross-company collaboration — the other party can only see the project you invited them to. All other projects in your organization remain completely isolated. No need to worry about information leaks.

## What Can Invited Members Do

Once invited, members have the following permissions:

| Mode | Available? | Description |
|------|-----------|-------------|
| **Discussion** | ✅ Immediately | Can freely post in the discussion area, @anyone and @Agent |
| **Private Agent** | ✅ Immediately | Can independently chat with AI, ask questions, validate content |
| **Team Agent** | ❌ Requires authorization | Must be **authorized by the project Owner** before use |

> The design philosophy: invited members can immediately **see, chat, and ask questions**, but to have the Agent **actively modify code**, the Owner must approve. This keeps collaboration open while preventing the chaos of "anyone can change the code."

## Additional Operations

Two more options at the bottom of the panel:

| Action | Description |
|--------|-------------|
| **Add Robot** | Add a robot to the project |
| **Disband Group** | Dissolve the current project group (use with caution) |
