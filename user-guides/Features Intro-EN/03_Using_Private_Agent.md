# Using Private Agent

## What Is Private Agent

Private Agent is a chat mode within the project group chat, primarily for **asking questions and planning**. When Team Agent is being used by a teammate, you don't have to sit idle — open Private Agent and keep researching, asking, and thinking without missing a beat.

## How It Differs from Team Agent

| Comparison | Team Agent | Private Agent |
|-----------|-----------|--------------|
| Mode | Coding mode — Agent writes code directly | Ask mode — questions and planning |
| Access Limit | One person at a time | No limit — everyone can use it simultaneously |
| Permission | Owner-controlled, can be revoked | No authorization needed |
| Use Case | Have the Agent execute development tasks | Research, validation, brainstorming |

> A typical scenario: your teammate is using Team Agent to develop a feature, while you're in Private Agent mapping out the next requirement. When they finish and you take over Team Agent, your plan is already clear — just start building. Efficiency doubles.

## Core Capabilities

### Contextual Understanding
The AI (Agent) shares the full context of your project with you:
- Global context sync: AI, developers, and product managers all work from the same repo, the same file structure, the same knowledge base
- No need to paste files manually or repeatedly explain "what this project does"
- New members can ask a few questions in Private Agent to get up to speed quickly — way faster than reading through docs

## Billing

- Private Agent usage is **also charged to the project Owner's quota**
- Because Private Agent conversations are considered work done for the project

## Message Actions

- Supports **Forward**: Private Agent tab forwarding has **no restrictions** (unlike Team Agent)
- Supports **quoting** and **copying** messages
