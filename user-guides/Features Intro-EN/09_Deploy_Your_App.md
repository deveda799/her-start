# Deploy Your App

## Preview

- Click the Preview button to preview your project directly
- You can also check available ports in Service first, then preview

> Always Preview before deploying. Confirm everything looks right so you don't discover a missing button after it's live.

## Deploy

### What It Means
- A port-free, **permanent deployment** with a link that never expires (and never changes)
- Think of it as **publishing a website**

### When to Use It
- When the project and pages are finalized (if you make changes later, the deployment auto-updates)
- Best suited for **static HTML and simple websites**
- **Not suitable for projects with backend databases**
- Recommended to Preview first before deploying

### Two Ways to Deploy

**Option 1: Deploy Tab (Bottom-Left)**
There are three tabs at the bottom-left of the page:

| Tab | Description |
|-----|-------------|
| **Service** | View available ports and service status |
| **Terminal** | Command-line terminal |
| **Deploy** | Click to enter the deployment flow (highlighted in red) |

Click the **Deploy** tab to start the deployment process.

**Option 2: Type "deploy" in Team Agent**
Simply type **deploy** in the Team Agent chat box, and the Agent will handle the entire deployment automatically.

> Yes, you read that right — just type "deploy" and the Agent takes care of everything. Once done, you'll get a permanent link that you can share with teammates, clients, or friends.

## Viewer: Middle Panel Preview Area

The file and project preview area is organized in two rows:

### Row 1 (Same row as tabs)
- Shows project-level **Preview** and **Deploy** buttons

### Row 2 (To the right of the file path / breadcrumb)
Different buttons appear based on file type:

| File / Project Type | Button Options |
|--------------------|----------------|
| **Markdown (.md)** | Source, Preview (toggle between raw markdown and rendered view) |
| **App projects** | My Device, Android, iOS, Web, Refresh (My Device supports Expo Go QR scanning for mobile preview) |
| **Git files with changes** | Preview, Diff View (shows before-and-after comparison) |
| **HTML files** | Source, Preview, New Tab |

## File Sharing

- HTML, JSON, Markdown, and other document formats all support **shareable links**

## Tab Management

- When you have multiple tabs open for previewing, **right-click** a tab to close it
- Three close options: Close Current, Close Others, Close All

## Comment Feature

- Select content within a project to **quote in Chat**, **add a Comment**, or **copy**
- Lines with Comments are highlighted in **red**
- Click the floating Comment icon to view all Comments (line numbers are highlighted in sync)

> Comments are perfect for code reviews or content proofreading — annotate directly on the relevant line instead of saying "there's an issue on line 42" in the chat. Way clearer.

## Markdown Editing

When editing Markdown documents:
- Select a line and **right-click** to access formatting and editing options
- Type "/" at the beginning of a line to access project and formatting commands
