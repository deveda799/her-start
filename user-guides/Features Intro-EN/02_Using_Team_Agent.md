# Using Team Agent

## What Is Team Agent

Team Agent is the core mode in a project's group chat — you talk directly to the AI Coding Agent, and it writes code and builds your project. Just describe what you want in natural language, and the Agent gets to work.

## Usage Rules

- Only **one person at a time** can talk to the Agent in Team mode
- Agent access is **controlled by the Owner** (project creator), who can revoke it at any time
- When multiple people request access simultaneously, the Owner approves one at a time; others wait in queue
- Costs are deducted from the **project Owner's** quota

> Having only one person at a time might sound limiting, but it's actually designed to prevent code conflicts. Other members can use Private Agent to do research and prep work in parallel. When it's their turn, they're ready to go — which often ends up being more efficient.

## Core Capabilities

### Visual Coding
Watch code changes happen in real time, like seeing cursors move in Figma:
- No waiting for builds
- No screenshots needed
- No meetings to explain "where are you editing right now?"

### Real-Time Sync
Multiple people working on the same repo without conflicts, branches, or sync delays:
- Changes appear instantly
- Commit status syncs across the team
- Every change is broadcast in real time

### Collaboration Trail
All discussions, edits, and code context are preserved in one space:
- No more "nobody knows the latest status"
- No miscommunication from scattered information
- The team stays aligned at all times

> If you've ever dealt with "someone changed the code but nobody knows who" or "we discussed it on Slack but nobody updated the docs" — Team Agent's collaboration trail solves exactly that. All context lives together naturally; no extra maintenance needed.

## Message Filter

There's a **Team Agent toggle** in the top-right corner of the chat window:
- **Off (default)**: Shows all messages, including the Agent's work process and code logs
- **On**: Hides the Agent's work process, showing only user messages and questions

Super useful when the chat history gets long — flip this toggle to quickly scan what team members discussed, skipping the code details in between.

## Message Actions

- **Restore**: Click a previous prompt to roll back to that state. Made a mistake? Go back to any earlier version instantly
- **Forward**: Forward messages to the current group or other groups
- Also supports **quoting** and **copying** messages
