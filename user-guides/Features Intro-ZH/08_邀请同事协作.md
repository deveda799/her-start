# 邀请同事协作

## 组织简介

- 注册时系统自动创建一个默认组织，组织角色只有 **Owner**（创建者）和 **Member**（成员）
- 每个项目都属于一个组织，不同组织的项目**互相不可见**
- 所有 Agent 权益跟随**项目 Owner** 扣除（包括 Team Agent 和 Private Agent）

## 如何打开成员管理

点击项目**右边栏的「人+」图标**（图标旁会显示当前成员数量），打开成员管理面板。

面板顶部有三个 tab：

| Tab | 说明 |
|-----|------|
| **Members** | 查看当前项目的所有成员，显示姓名、头像和邮箱，Owner 会有标识 |
| **Invite** | 邀请新成员加入项目 |
| **Add Agent** | 添加你之前创建的 AI Agent 作为项目成员，让 AI 伙伴参与协作 |

## 邀请方式

| 方式 | 操作 | 效果 |
|------|------|------|
| **组织内添加** | Invite tab 中搜索 Organization Contacts，勾选后点击 **Invite** | 组织内成员加入当前项目 |
| **邀请外部成员** | Invite tab 底部点击 **「Add external members」** 按钮 | 外部成员**只加入当前项目，不加入组织** |
| **添加 AI Agent** | Add Agent tab 中选择已创建的 Agent | AI 伙伴加入项目参与协作 |

> 邀请外部成员特别适合跨公司合作的场景——对方只能看到你邀请的那个项目，组织里的其他项目完全隔离，不用担心信息泄露。

## 邀请进来后能做什么

被邀请的成员进入项目后，权限如下：

| 模式 | 是否可用 | 说明 |
|------|---------|------|
| **Discussion** | ✅ 直接可用 | 可以在讨论区自由发言、@ 任何人和 Agent |
| **Private Agent** | ✅ 直接可用 | 可以独立和 AI 对话、提问、验证内容 |
| **Team Agent** | ❌ 需要授权 | 必须获得**项目创建者（Owner）的授权**后才能使用 |

> 设计思路是这样的：邀请进来就能**看、聊、问**，但想让 Agent **动手改代码**，必须 Owner 点头。这样既保证了协作的开放性，又不会出现"谁都能改代码"的混乱。

## 其他操作

面板底部还有两个操作：

| 操作 | 说明 |
|------|------|
| **Add Robot** | 添加机器人到项目 |
| **Disband Group** | 解散当前项目群组（谨慎操作） |
