# 群聊协作

## 项目与群聊的关系

- 每一个项目对应一个群聊
- 每一个群聊对应三种聊天模式：Team Agent、Private Agent、Discussion

## 三种聊天模式

| 模式 | 定位 |
|------|------|
| **Team Agent** | 团队与 AI 协作，产出代码和内容 |
| **Private Agent** | 个人 AI 助手，独立提问和验证 |
| **Discussion** | 团队成员之间的开放讨论区 |

## 协作流程

三种模式配合使用，形成完整的团队协作闭环：

1. **Team Agent 产出内容** → 群里所有成员都能看到 Team Agent 的对话记录和产出
2. **Discussion 讨论反馈** → 成员看到产出后，在 Discussion 中讨论、提意见
3. **Private Agent 独立验证** → 成员可以在 Private Agent 中对产出进行验证、测试或深入提问，不影响 Team Agent 的工作

> 简单说就是：**Agent 干活 → 大家讨论 → 各自验证**，三个模式各司其职。不需要再开额外的会议来同步进度，因为所有人都在同一个信息流里。

## Discussion 详细介绍
- 项目内的**开放讨论区**，全部成员均可参与
- 可以针对项目进行自由聊天
- 支持 **@ 任何人**，也可以 **@ Agent**（对，你可以在讨论区直接召唤 AI）
- 鼠标悬停在单条消息上，会出现三个操作按钮（从左到右）：

| 按钮 | 图标 | 作用 |
|------|------|------|
| **回复** | ↩ 左弯箭头 | 针对这条消息进行回复，形成消息引用 |
| **复制** | ⧉ 重叠矩形 | 复制该条消息内容 |
| **转发** | ↗ 外指箭头 | 将该条消息转发给其他人或群聊 |

## 邀请成员

点击项目**右边栏的「人+」图标**，打开成员管理面板：
- **Members** tab：查看当前成员
- **Invite** tab：邀请组织内成员或外部好友加入项目
  - 外部好友**只加入当前项目，不加入组织**
  - 加入组织需用户自己在左上角通讯录 Contacts 中点击「Join Organization」
- **Add Agent** tab：添加已创建的 AI Agent 参与协作

## Chat（真人聊天）

左侧栏 Project 旁边的 **Chat** tab，点击进入真人聊天区域：
- 新增「New Chat」，可以新建真人私聊（1v1）、真人群聊
- 人员进群退群会有**广播通知**
- 消息动作支持**复制、引用、转发**
- 支持搜索组织 Contacts 和 chat 记录发起聊天

## 未读消息与 @提及

- Chat、Notification、Projects 均会展示未读消息、@提及消息状态
- **@提及消息的展示优先级高于未读消息**，有人 @ 你的时候不会被埋没

## Notification（通知）

- 展示全部系统通知、agent 权限通知、组织加入、好友申请、充值通知等
- 包括「View in Project」、「Upgrade」快捷跳转，方便直接定位到对应项目
