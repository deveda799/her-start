# 部署应用

## Preview（预览）

- 可以直接点击 Preview 按钮进行项目预览
- 也可以先查看 Service 中可用的端口，再进行项目预览

> 部署之前建议先 Preview 看一眼效果。确认没问题再部署，省得上线后发现少了个按钮又得改。

## Deploy（部署）

### 含义
- 不用端口，**永久部署**的，不会失效的链接（链接也不会变）
- 可以理解为类似**网站的发布**

### 适用场景
- 确定好项目和页面没有修改（如果修改，会自动更新部署）
- 适合**静态 HTML、简单小网站**这一类的项目
- **后台数据库类型的项目不适用**
- 最好先通过 Preview 进行 check

### 两种部署方式

**方式一：左下角 Deploy tab**
页面左下角有三个 tab：

| Tab | 说明 |
|-----|------|
| **Service** | 查看可用端口和服务状态 |
| **Terminal** | 终端命令行 |
| **Deploy** | 点击进入部署操作（红色高亮） |

点击 **Deploy** tab 即可进入部署流程。

**方式二：Team Agent 对话框直接输入**
在 Team Agent 对话框中直接输入 **deploy**，Agent 会自动执行部署。

> 对，你没看错——直接打一个"deploy"就行，Agent 会处理剩下的一切。部署完成后会给你一个永久链接，可以直接发给同事、客户、朋友。

## Viewer 中间栏预览区域

文件和项目预览区域分为两行：

### 第一行（与 tab 区域同一行）
- 展示项目层级的 **Preview** 和 **Deploy**

### 第二行（文件路径/面包屑右侧）
根据不同文件类型显示不同按钮：

| 文件/项目类型 | 按钮选项 |
|---------------|----------|
| **Markdown (.md)** | Source、Preview（展示 markdown 原格式和预览模式） |
| **App 项目** | My Device、Android、iOS、Web、Refresh（其中 My Device 支持 Expo Go 扫码进行移动端预览） |
| **Git 有 changes 的文件** | Preview、Diff View（展示修改对比模式） |
| **HTML 文件** | Source、Preview、New Tab |

## 文件分享

- HTML、JSON、Markdown 等格式文档均支持**分享链接功能**

## Tab 管理

- 打开多个 tab 进行预览，可以在 tab 上**右键**进行关闭
- 三种关闭选项：Close 当前、Close 其他、Close All

## Comment 功能

- 项目内内容选中后支持**引用到 Chat**、**添加 Comment**、**复制**
- 添加 Comment 的内容行号会展示为**红色**
- 点击悬浮的 Comment icon 可以查看全部已添加的 Comment（行号会同步高亮）

> Comment 特别适合代码审查或内容校对——直接在对应行上标注意见，比在聊天里说"第 42 行那个地方有问题"清晰得多。

## Markdown 编辑

Markdown 格式的文档，在编辑状态下：
- 支持选中一行后，点击**右键**进行格式等相关编辑
- 支持在行前输入「/」进行项目、格式编辑
