# 上传文件和知识库

## 文件和文件夹操作

CodeBanana 的文件管理功能在 Studio 页面左侧的项目列表中。

### 如何上传

在项目或文件夹上点击 **「…」**：

| 操作 | 说明 |
|------|------|
| **Upload file** | 上传单个文件 |
| **Upload folder** | 上传整个文件夹 |

> 一级目录（项目）和二级目录（文件夹）均支持上传，在对应层级点击「…」即可。

上传完成后，这些文件就自动成为项目知识的一部分。Agent 可以直接读取和理解它们——你不需要手动贴内容，也不用解释文件是什么，直接说"帮我分析刚才上传的那份报告"就行。

### 上传限制

| 限制项 | 上限 |
|--------|------|
| **单个文件大小** | 不超过 **100MB** |
| **图片数量** | 最多 **10 张** |
| **总文件数量（含图片）** | 最多 **50 个** |

### 支持的文件格式

基本上你能想到的格式都支持：

| 类别 | 常见格式 |
|------|---------|
| **代码** | .js、.ts、.py、.java、.go、.cpp、.c、.rs、.rb、.php、.swift、.kt 等 |
| **前端** | .html、.css、.scss、.less、.jsx、.tsx、.vue、.svelte 等 |
| **文档** | .md、.txt、.pdf、.doc、.docx、.ppt、.pptx、.xls、.xlsx、.csv 等 |
| **图片** | .png、.jpg、.jpeg、.gif、.svg、.webp、.ico、.bmp 等 |
| **音频** | .mp3、.wav、.aac、.ogg、.flac、.m4a 等 |
| **视频** | .mp4、.mov、.avi、.webm、.mkv 等 |
| **配置** | .json、.yaml、.yml、.toml、.xml、.env、.ini 等 |
| **设计** | .figma、.sketch、.psd、.ai 等 |
| **压缩包** | .zip、.rar、.tar、.gz、.7z 等 |

### 其他文件操作

| 操作 | 说明 |
|------|------|
| **Rename** | 重命名文件或文件夹 |
| **New File / New Folder** | 新建文件或文件夹 |
| **Copy Path** | 复制相对路径（相对于项目根目录） |
| **Download** | 下载文件或文件夹 |
| **Duplicate to** | 复制到其他位置 |
| **Move to** | 移动到其他位置 |
| **Delete Project** | 删除项目（需拥有 Agent 权限） |

## 上下文共享

这是 CodeBanana 最强大的机制之一：上传到项目中的所有文件，自动成为整个团队和 AI 的共享知识。

- AI 和团队成员都基于同一个 repo、同一批文件、同一份知识工作
- 不需要反复贴文件、不需要手动同步
- 新人加入项目后，所有历史资料都在，不用从零交接

> 想象一下：产品经理上传了需求文档，设计师上传了设计稿，开发直接让 Agent 基于这些材料写代码——所有人看到的都是同一份真相，再也不会出现"你看的是哪个版本？"的问题。

## 信息转发（Forward 功能）

- AI 会将多人讨论的上下文融入到后续 coding，减少重复沟通
- 支持单独、批量转发到不同群、不同 agent、Discussion tab

## 设计到开发的全链路上下文共享（Preview 功能）

- 设计师能理解正在开发什么
- PM 能看到逻辑文件和变更
- QA 能直接定位代码实现
