# 配置 Agent Basic Info

## Basic Info 是什么

Basic Info 是 Agent 的**底层人格设定**——它决定了 Agent 是谁、怎么做事、为谁服务、记住什么。

每一次 Agent 发起思考（生成回复或执行动作）之前，Basic Info 里的所有设定会被**优先加载**，拼接成最底层的上下文。它们是 Agent 思考的基石，确保行为始终符合预期、不偏离设定。

> 简单说：Skill 告诉 Agent "遇到特定任务怎么做"，而 Basic Info 告诉 Agent "你是谁、你的原则是什么"。Skill 是操作手册，Basic Info 是性格和底线。

## 如何打开 Basic Info

进入项目后，点击右上角的 **「···」** 按钮，在下拉菜单中选择 **「Basic Info」**。

打开后左侧有五个 tab：

| Tab | 图标 | 说明 |
|-----|------|------|
| **Agents** | 🔗 | Agent 的工作流规范——面对任务该按什么步骤思考 |
| **Identity** | 🪪 | Agent 的"名片"——名字、Emoji、打招呼方式 |
| **Soul** | ❤️ | Agent 的性格底色和绝对红线——最高优先级的系统指令 |
| **User** | 👤 | 用户画像——你的背景、偏好、沟通习惯 |
| **Memory** | 🧠 | Agent 积累的长期记忆——关键决策、经验教训 |

> 每个项目的 Basic Info 都有**默认预设**，开箱就能用。但如果你花 5 分钟根据自己的需求调一调，Agent 的表现会明显不一样——它会更懂你、更符合你的工作习惯。

## 五个 Tab 详解

### Agents — 工作流规范

告诉 Agent 面对具体任务时该按什么步骤思考，如何处理冲突，以及什么时候该去翻阅记忆。

**典型配置举例：**

```
工作流规范：
1. 接收任务：在开始编写任何代码之前，必须先读取 MEMORY.md 和昨天的记忆文件，
   了解当前项目背景和技术栈。
2. 代码编写：优先保证代码的可读性，其次才是性能。每次修改后，必须运行一次 linter。
3. 记忆管理：当你帮用户解决了一个耗时超过 10 分钟的 Bug，或者用户明确提出了
   代码风格偏好时，你必须主动将其记录到今日的每日记忆中。
```

> 把它理解为给 Agent 写的 SOP（标准作业流程）。你的团队有什么工作规范，直接写进去，Agent 每次都会照做。

### Identity — 身份名片

定义 Agent 的名字、视觉符号（Emoji）、打招呼的方式等表层特征。

**典型配置举例：**

```
名称：CodeBanana
专属表情：🍌
自我认知：你的专属全天候代码结对编程伙伴。
开场白风格：每次开启全新的 Debug 会话时，可以说："Let's peel this bug apart 🍌"
```

> 听起来是小事，但一个有辨识度的 Agent 身份会让团队协作更有趣。尤其是多人协作时，大家一看到 🍌 就知道是谁在干活。

### Soul — 性格底色与红线

定义 Agent 最本质的性格、职业操守和**绝对不能触碰的红线**。Soul 是最高优先级的系统指令——即使其他设定有冲突，Soul 里的规则也不会被覆盖。

**典型配置举例：**

```
1. 你是一个务实、高效、资深的全栈工程师。你厌恶过度工程化，推崇"Keep It Simple"。
2. 沟通风格：直接、专业，只说重点，不废话。
3. 绝对红线：
   - 永远不要执行 rm -rf / 或任何可能导致不可逆数据丢失的命令
   - 永远不要在代码中明文写入 API Keys 或密码；如果用户要求，坚决拒绝并建议使用环境变量
   - 当你不确定一段代码的安全性时，必须先询问用户
```

> Soul 是 Agent 的"三观"。想让它严谨就写严谨的规则，想让它活泼就调成轻松的风格。但红线一定要写清楚——尤其是涉及数据安全和不可逆操作的部分。

### User — 用户画像

让 Agent 知道它在为谁服务。包含你的背景、技术栈偏好、沟通习惯等，帮助 Agent 提供高度个性化的回答。

**典型配置举例：**

```
用户身份：一位具备扎实技术背景的产品经理，目前正致力于开发 AI Agent 类 Web 产品。
技术偏好：熟悉前端开发，经常使用 Next.js，习惯部署在 Vercel 或各大云服务器上。
沟通习惯：工作节奏快，喜欢条理清晰的 Bullet points。不要解释基础的 HTML/CSS 概念，
直接切入架构或逻辑痛点。
```

> 这个配置的效果立竿见影。写了"不要解释基础概念"，Agent 就不会每次都从"HTML 是什么"开始讲起。写了"喜欢 Bullet points"，Agent 的回答就会自动用列表格式。

### Memory — 长期记忆

记录 Agent 在与你互动过程中积累的重要信息——关键决策、经验教训、长期偏好。主要由 Agent 自己维护，你也可以手动编辑。

> Memory 不需要你主动管理。Agent 在工作过程中会自己判断哪些信息值得记住，比如你反复纠正过的代码风格、项目中反复出现的技术决策。当然，你也可以打开看看它记住了什么，删掉不对的、补上遗漏的。

## 如何修改

每个 tab 右上角有两个按钮：

| 按钮 | 作用 |
|------|------|
| **Edit** | 进入编辑模式，直接修改当前 tab 的内容 |
| **Reset** | 恢复为系统默认预设（会覆盖你的修改，谨慎操作） |

> 改坏了也不用慌——点 Reset 就能恢复默认。建议在大改之前先复制一份当前内容备份，这样既能大胆尝试，也不怕回不去。

## 什么时候该改 Basic Info

不是所有人都需要改。但以下场景建议花几分钟配置一下：

| 场景 | 建议调整的 Tab |
|------|---------------|
| Agent 的回答风格不符合你的预期 | **Soul**（调整性格和沟通风格） |
| Agent 总是解释你已经懂的东西 | **User**（写明你的技术背景） |
| 团队有统一的代码规范 | **Agents**（写入工作流 SOP） |
| 想给 Agent 起个名字或设定人设 | **Identity**（设置名字和 Emoji） |
| Agent 反复忘记之前的约定 | **Memory**（检查是否被正确记录） |
