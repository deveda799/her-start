# 使用 Team Agent

## Team Agent 是什么

Team Agent 是项目群聊中的核心模式——直接和 AI Coding Agent 对话，让它帮你写代码、做项目。你只需要用自然语言描述想要什么，Agent 就会动手执行。

## 使用规则

- 每次 team 中**只能有一个人**和 Agent 对话
- Agent 权限由**群主（Owner）控制**，群主可以随时收回
- 多人同时申请时，群主逐个审批，其他人排队等待
- 费用从**项目 Owner** 的额度扣除

> 一次只能一人操作听起来有限制，但其实这是为了保证代码不冲突。其他成员可以在 Private Agent 里同步做调研、验证，等轮到自己时直接上手，效率反而更高。

## 核心能力

### 可视化编码
实时看到代码变化，像 Figma 里看光标移动一样：
- 无需等待构建
- 无需截图
- 不需要开会解释"你现在在哪里改？"

### 实时同步
多人同一个 repo 协作，没有冲突、分支、延迟引起的混乱：
- 即写即显
- 提交状态同步
- 变更实时广播给团队

### 协作轨迹
所有讨论、修改、代码上下文都在一个空间内沉淀：
- 不会出现"没人知道最新状态"
- 不会因为信息分散导致误解或返工
- 团队时刻保持 aligned

> 如果你以前遇到过"代码改了但不知道谁改的"、"Slack 里讨论完但没人更新文档"这类问题，Team Agent 的协作轨迹就是为了解决这个。所有上下文天然在一起，不需要额外维护。

## 消息过滤

聊天窗口右上角有一个 **Team Agent 开关**（toggle 按钮）：
- **关闭（默认）**：显示所有消息，包括 Agent 的工作进程和代码记录
- **打开**：屏蔽 Agent 的工作过程，只保留用户的消息和提问记录

聊天记录很长的时候特别好用——打开这个开关，快速浏览团队成员聊了什么，跳过中间的代码细节。

## 消息动作

- **Restore**：点击之前发的 prompt，可以回退到那个节点的状态。改坏了不怕，随时回到之前的版本
- **Forward**：转发消息到当前群或其他群
- 还支持**引用、复制**消息
