# 权益和用量管理

## 套餐与定价

CodeBanana 提供四种套餐：

| 套餐 | 月费 | 年付（/月） | 包含 Requests | 适用场景 |
|------|------|------------|--------------|---------| 
| **Free Plan** | $0/month | $0/month | 30 requests | 探索和体验，个人爱好项目 |
| **Personal Plan** | $20/month | $16/month | 300 requests | 个人开发者，日常开发使用 |
| **Professional Plan** | $40/month | $32/month | 1000 requests | 专业开发者，高频使用 |
| **Team Plan** | 自定义 | 自定义 | 每人 300 requests/月 | 团队协作，企业级需求 |

> Free Plan 的 30 requests 足够你完整体验一个小项目了——从创建到对话到部署，走一遍全流程再决定要不要升级，完全来得及。

### VM 配置选项
- Personal 和 Professional 套餐可以根据需求选择**不同的虚拟机规格**
- 账单会根据所选虚拟机服务进行扣费
- Usage 和 Upgrade 页面也可以随时升级虚拟机

---

## 非 Team Plan（Free / Personal / Professional）

### 计费方式

套餐内包含固定数量的 requests（30 / 300 / 1000）。用光之后有两个选择：

1. **升级套餐**：在 Upgrade 页面切换到更高级的套餐
2. **按用量计费**：在 Usage 页面启用按用量计费模式——设定一个金额限度，系统会自动通过 **Stripe** 拉起账单进行充值

> 不用担心突然被断——用量快到的时候会有提示。开启按用量计费后，手头的工作可以继续推进，不会因为 requests 用完就卡住。

### 扣费机制

- 扣费归属和 Agent 归属保持一致——项目的 **Owner** 可以管理 Agent 权限，扣的费用也从 Owner 的额度扣除
- **Private Agent 权益**也从项目 Owner 账户扣除（因为 Private Agent 对话也被认为是为项目进行的工作）

> 简单说就是：**谁创建的项目，谁买单**。不管是你自己用 Team Agent 还是同事在你项目里用 Private Agent，都从你（Owner）的额度里扣。

## Team Plan

Team Plan 面向需要团队协作的组织，由**管理员（Owner）统一管理**成员的权益和用量。

### 计费方式

- 每位被添加到 Team Plan 的成员，每月享有 **300 requests**
- 管理员可以为每位成员选择不同的虚拟机配置，账单根据虚拟机档位从 Owner 的信用卡扣除席位费
- 套餐内 requests 用光后，会自动启用按用量计费模式，**额度上限由管理员设置**

### 管理员操作

#### 添加成员到 Team Plan

1. 进入个人中心 → **Usage** → **Members** 页面
2. 点击 **「Add Member」**，从已加入组织的成员中选择
3. 设置该成员的**虚拟机配置**和 **On-demand Usage 限额**
4. 确认后，席位费从 Owner 的信用卡中扣除

> 加入组织 ≠ 加入 Team Plan。成员加入组织后，还需要管理员在 Members 页面手动添加到 Team Plan，才能享受团队权益。没有被添加的成员，扣费逻辑按照非 Team Plan 从该成员自身的套餐扣除。

#### 更改成员配置

管理员可以随时调整成员的虚拟机配置和 On-demand Usage 限额：

| 操作 | 生效时间 | 费用处理 |
|------|---------|---------|
| **升级虚拟机** | 立即生效 | 差价从 Owner 信用卡扣除 |
| **降级虚拟机** | 立即生效 | 余额暂存在 Stripe 账户，用于抵扣后续账单 |

**差价计算方式：**
- 使用额度 = 总价值 ÷ 总天数 × 使用天数 与 $0.15/request，两者取**较高值**
- 余额 = 已支付费用 - 使用额度
- 差价 = 新套餐价格 - 余额

> 降级后的余额会自动留在 Stripe 账户抵扣下次账单。如果需要退回这笔金额，需要发送邮件申请。

#### 查看账单

管理员可以在个人中心 → **Billing** 页面查看所有账单明细。

### 成员视角

被管理员添加到 Team Plan 后：
- 每月享有 **300 requests**
- 虚拟机配置和 On-demand Usage 限额由管理员设定
- **用至上限后，需要联系管理员提高限额**

> 作为成员你不需要操心费用问题——额度和配置都由管理员统一管理。用量快到上限的时候，找管理员说一声就行。

### Team Plan 扣费机制

| 扣费时机 | 说明 |
|---------|------|
| **开通席位时** | 从 Owner 信用卡扣除席位费 |
| **自动结算** | 所有 Team Plan 成员的 On-demand Usage 累计达到 **$2,000**，或时间周期达到 **30 天**时，自动从 Owner 信用卡扣费 |

## 查看用量

点击页面**右上角头像**进入个人中心：

| 页面 | 说明 |
|------|------|
| **Usage** | 查看全部 token 和权益用量情况，支持导出 |
| **Members & Quotas** | 查看全部组织成员的用量情况（Team Plan 管理员可见） |
| **Billing** | 查看所有账单明细（Team Plan 管理员可见） |

每一笔消耗都有明细，花了多少、花在哪里，清清楚楚。

## 通讯录管理

- **Contacts**：查看组织申请记录、组织成员、添加的好友
- 目前只能展示 Studio 页面选中的组织，其他组织需要切换查看
