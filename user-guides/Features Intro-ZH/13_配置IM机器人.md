# 配置 IM 机器人（Channel）

## Channel 是什么

Channel 让你把项目里的 Agent 接入日常使用的 IM 软件——在飞书、钉钉、企业微信、Slack 等平台上直接和 Agent 对话，不用切到 CodeBanana 页面。

支持接入的 IM 平台：

| 平台 | 说明 |
|------|------|
| **Feishu（飞书）** | 国内团队常用 |
| **DingTalk（钉钉）** | 国内团队常用 |
| **Wecom（企业微信）** | 国内团队常用 |
| **QQ** | 适合轻量级场景 |
| **Slack** | 海外团队常用 |
| **Discord** | 社区和开发者团队常用 |
| **Telegram** | 海外个人和小团队常用 |
| **Mattermost** | 自托管 IM 方案 |

> 接入之后的效果是：你在飞书群里 @ 机器人说一句话，CodeBanana 里的 Agent 就会响应并执行。定时任务的结果、数据预警、周报这些也会直接推到 IM 群里，不用打开 CodeBanana 就能收到。

## 如何打开 Channel 配置

有两个入口，选哪个都行：

| 入口 | 路径 |
|------|------|
| **项目内** | 进入项目 → 右上角 **「···」** → 选择 **「Channels」** |
| **个人中心** | 右上角头像 → 个人中心 → **「IM Setting」** |

> 区别在于：项目内的入口可以直接绑定 Bot 到当前项目；个人中心的入口适合统一管理你创建的所有 Bot。

## 创建 Bot

1. 点击 Agent Config → **Channel** → **「+ Add Bot」**（如果是第一次，按钮显示为「+ Add Your First Bot」）
2. 选择要接入的 **IM 平台**
3. 填写该平台对应的 **ID** 和 **Secret**（每个平台的获取方式不同，按平台文档操作即可）
4. 点击确认，完成创建

创建后 Bot 会显示连接状态：

| 状态 | 含义 |
|------|------|
| **Connected** | ID 和 Secret 填写正确，Bot 已成功连接 |
| **Disconnect** | 信息填写有误或连接失败，需要检查并重新配置 |

> 同一个 IM 平台可以创建多个 Bot。比如你在飞书有多个群需要接入不同项目的 Agent，就创建多个飞书 Bot 分别绑定。

## 绑定 Bot 到项目

创建好 Bot 之后，还需要把它绑定到具体的项目才能生效：

1. 进入项目 → 右上角 **「···」** → **「Channels」**
2. 选择要绑定的 Bot，点击开关**打开**即可完成绑定

### 绑定规则

| 规则 | 说明 |
|------|------|
| **一个项目只能绑一个 Bot** | 确保消息不会混乱 |
| **一个 Bot 只能绑到一个项目** | 已被绑定的 Bot 不会出现在其他项目的可选列表中 |
| **只有 Connected 状态的 Bot 可以绑定** | Disconnect 状态的 Bot 需要先修复连接 |

> 简单说就是一对一关系：一个项目配一个 Bot，一个 Bot 服务一个项目。如果你有 3 个项目都需要接入飞书，那就创建 3 个飞书 Bot，分别绑定。

## 完整流程总结

```
创建 Bot：Agent Config → Channel → Add Bot → 选平台 → 填 ID 和 Secret → 确认创建

绑定到项目：Agent Config → Channel → 选择 Bot → 打开开关 → 绑定完成

在 IM 中使用：在对应 IM 平台的群里 @ 机器人 → Agent 响应并执行
```
