# 查看和使用看板

## 看板是什么

组织 AI 效率指数看板是组织级的数据面板，**全面监控组织 AI 构建进展与效能指标**。

- 组织 Owner 可以判断 AI 转型进度，获取优化建议
- Member 可以看到自身 AI 能力成长和排名

> 一句话：这个看板告诉你，你的团队用 AI 用得怎么样——谁在用、用了多少、产出了什么、还有哪里可以提升。

## 如何进入看板

在 CodeBanana 页面**右上角点击头像**进入个人中心，点击「**AI-Native Index Dashboard**」进入。

## 看板顶部：筛选和操作栏

顶部有四个操作区域：

| 按钮 / 控件 | 作用 |
|------------|------|
| **组织选择器** | 切换要查看的组织（如 Mobvoi） |
| **选择部门** | 筛选特定部门的数据 |
| **日期筛选** | 选择时间范围（如 2026/03/05 - 2026/03/11） |
| **EN / 中文** | 右上角语言切换，默认英文，支持切换为中文 |

## 核心数据卡片（三张）

筛选栏下方展示三张核心数据卡片：

| 卡片 | 含义 | 示例 |
|------|------|------|
| **活跃成员** | 组织总人数和目前活跃人数 | 168 人，活跃 118 人（70%） |
| **Token 总消耗** | 组织在选定时间内的 Token 总用量和人均用量 | 5.8B Tokens，人均 34.3M |
| **总投入成本** | 组织共投入成本和全员人均成本 | $29,063，人均 $172.99 |

## AI 成熟度雷达图

雷达图直观展示组织的五大 AI 维度评分（满分 100），一眼看出哪个维度强、哪个维度弱：

| 维度 | 缩写 | 衡量什么 | 核心指标 |
|------|------|---------|---------| 
| **AI First（AI 渗透度）** | AIF | AI 在日常工作中的渗透程度 | Token 消耗总量、Token 人均消耗、AI 生成代码占比 |
| **超级个体指数** | SII | "一人 + AI"独立完成项目的能力 | 超级个体占比 |
| **端到端质量延展能力** | EPC | 项目从需求到交付是否形成闭环 | 延展覆盖率 |
| **组织扁平化指数** | OFI | 组织结构是否扁平高效 | 平均项目参与人数、组织效率占比 |
| **Agent 协作表现度** | ACM | 与 AI Agent 的协作深度 | Team Agent 使用率、多 Agent 协同率 |

> 第一次看到雷达图不用紧张，分数低不代表做得差——它更像是一个方向盘，帮你看清哪些方面还有提升空间。下面会具体说每个维度低了该怎么办。

## Token 使用趋势图

折线图展示选定时间范围内**组织总量**的 Token 使用趋势：
- 横轴为日期，纵轴为 Token 数量
- 鼠标悬停可查看某天的具体数值（如 03/09: 958.8M Tokens）
- 用来判断组织 AI 使用是在增长还是停滞

## 部门明细表

看板下方按部门拆解详细数据，每行展示一个部门：

| 列名 | 含义 |
|------|------|
| **部门名称** | 部门名及人数 |
| **请求量** | 该部门发起的 AI 请求次数 |
| **Token** | 该部门的 Token 消耗量和人均 Token |
| **费用** | 该部门的消耗金额和人均费用 |

- 支持排序，快速找到使用量最高或最低的部门
- 底部点击「**查看全部 XX 个部门**」展开所有部门

## 五大维度详情卡片

雷达图下方有五张详情卡片，逐一展示每个维度的**具体得分和明细指标**：

| 卡片 | 得分示例 | 展示的明细 |
|------|---------|-----------| 
| **AI First (AIF)** | 96 | Token 消耗总量、Token 人均消耗、单个产出项目平均消耗 |
| **超级个体指数 (SII)** | 50 | 成功部署项目的成员在活跃人数中的占比 |
| **组织扁平化指数 (OFI)** | 90 | 平均项目参与人数、组织效率占比 |
| **端到端质量延展能力 (EPC)** | 60 | 成功部署产出的项目占比 |
| **Agent 协作表现度 (ACM)** | 47 | Team Agent 使用率、多 Agent 协同率 |

## AI 智能分析报告

看板底部自动生成 **AI 智能分析报告**（约 10~30 秒），基于当前数据给出组织诊断和优化建议。

> 不用自己分析数字。看板会直接告诉你"哪里做得好、哪里需要改、下一步建议做什么"——拿着这份报告去汇报，省下好几个小时的整理时间。

## 看到低分怎么办？

| 低分维度 | 建议操作 |
|---------|---------| 
| **AI 渗透度低（AIF）** | 设定 AI 使用最低标准；需求先给 Agent 再给人；设置 3 个月 AI 代码占比目标 |
| **超级个体占比低（SII）** | 鼓励单人 + Agent 完成项目；共享 Prompt 模板；开展 Agent 使用培训 |
| **端到端能力弱（EPC）** | 推动项目从需求到部署的完整闭环；减少中间交接环节 |
| **组织扁平化低（OFI）** | 减少管理层级；鼓励跨部门直接协作 |
| **Agent 协作弱（ACM）** | 推动 Team Agent + Private Agent 同时使用；建立多 Agent 协作流程 |
| **Token 多但产出少** | 用更结构化的 Prompt；先让 Agent 输出计划再生成代码；逐步拆解任务 |
