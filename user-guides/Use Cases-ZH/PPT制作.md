# PPT 制作的 CodeBanana 使用指南

> 做 PPT 最痛苦的不是最终呈现，而是中间过程——对着空白页面不知道写什么、每页内容想半天、做完 PPT 还要准备演讲稿。CodeBanana 帮你把这三个环节串起来：先理清结构，再填充内容，最后写好讲稿。整个过程从"一天"压缩到"1 小时"。

---

## 场景一：5 分钟列好大纲

### 什么时候会用到

- 要做汇报 PPT 但不知道怎么组织结构
- 知道要讲什么，但不知道先讲什么后讲什么
- 需要快速确认 PPT 框架，再开始填内容

### 怎么做

如果你还没想好怎么组织，可以先在 **Private Agent** 里自己理一理——"我这次汇报要讲什么、重点在哪、老板最关心什么"。等思路清楚了，再让 Agent 正式出大纲。

告诉 Agent 你的场景：

> "下周要做 Q1 工作汇报 PPT，给老板看的，大概 20 分钟"

Agent 5 分钟出完整大纲：

- 建议 15 页，时间分配到每个部分
- **第一部分（4 分钟）**：成果总览——核心指标、重点项目、团队亮点
- **第二部分（8 分钟）**：重点项目复盘——每个项目的背景→做法→成果→经验
- **第三部分（3 分钟）**：问题与挑战——坦诚面对 + 改进措施
- **第四部分（4 分钟）**：Q2 规划——量化目标 + 项目计划 + 资源需求
- **结尾（1 分钟）**：一句话总结 + Q&A

觉得结构要调整？

> "项目只有两个，去掉项目 C，加个团队建设部分"

30 秒更新。大纲确认后再往下走，避免做到一半推翻重来。

## 场景二：20 分钟填完所有页面内容

### 什么时候会用到

- 大纲有了，但每页该放什么内容想不出来
- 想直接生成一个完整的 PPT，后续在上面改
- 被催着要 PPT，需要最快速度出一版

### 怎么做

> "按刚才的大纲帮我生成 PPT 内容"

Agent 逐页生成：

**封面页：** 标题 + 部门 + 姓名 + 日期

**项目复盘页：**
```
618 活动项目

背景：公司首次大型促销活动
目标：GMV 突破 100 万

做法：
→ 提前 2 周预热，覆盖 3 个渠道
→ 活动期间每日数据监控 + 实时调优
→ 阶梯优惠拉高客单价

成果：
GMV ¥128 万（超额 28%）| 新客占比 42% | ROI 3.2

经验：预热期投入直接影响活动首日爆发力
```

**问题与挑战页：** 列出问题 + 已采取措施 + 后续计划，而不只是列问题

最终输出完整的 PPT 文件或 HTML 演示文稿。如果是 HTML 版本，可以直接 **Deploy** 拿到永久链接——发给同事提前预览、发给老板在手机上看效果，都很方便。链接会自动更新，你改了内容不用重新发。

数据需要改？

> "第 4 页 GMV 改成 115 万"

改完更新。

## 场景三：10 分钟写好演讲稿

### 什么时候会用到

- PPT 做好了但不知道每页该怎么讲
- 担心超时或者讲得太干巴巴
- 需要控制节奏——什么时候停顿、什么时候看听众

### 怎么做

> "PPT 做好了，帮我写 20 分钟的演讲稿"

Agent 输出逐页演讲稿，不只是内容，还包括演讲技巧提示：

**封面页（30 秒）：**
> "各位好，接下来由我汇报市场部 Q1 的工作成果。先说结论：Q1 我们超额完成了 3 个核心指标中的 2 个，有一个没达标，但已经找到原因和改进方案。"

**核心指标页（1 分 30 秒）：**
> "先看核心数据。（指向第一个数字）GMV 完成 850 万，完成率 94%…"
> 
> ⏱ 提示：在这页不要展开细节，先给全局印象

**项目复盘页（3 分钟）：**
> "重点讲 618 项目…关键做对了三件事…（看向老板）如果 Q2 还有大促，这个打法可以复用。"
> 
> ⏱ 提示：多讲"为什么这么做"，少讲流水账

觉得太长？

> "压缩到 15 分钟"

Agent 精简话术，控制时间。

## 给做 PPT 的人的小建议

1. **Private Agent 先理思路**：不知道怎么组织的时候，先在 Private Agent 里跟 Agent 聊聊。它会帮你梳理逻辑，等你想清楚了再正式出大纲。

2. **一页一个核心信息**：Agent 生成的内容如果太密，让它精简——"这页只保留最重要的一个结论"。好的 PPT 不是信息越多越好。

3. **HTML 版本部署后当场演示**：如果生成的是 HTML 演示文稿，Deploy 后拿到链接，打开浏览器全屏就能讲。不用带 U 盘，不用担心电脑兼容性。

4. **演讲稿要练**：Agent 写的稿子帮你理清思路和节奏，但最终还是要自己练两遍。尤其是衔接句和停顿点，练过和没练过差别很大。

5. **用 Basic Info 固定汇报风格**：在 Soul 里写上"PPT 内容风格：一页一个核心信息，多用数据和图表，少用大段文字。演讲稿风格：口语化、有节奏、标注停顿点"。在 User 里注明"汇报对象通常是 CEO/VP 层，关注结果和建议，不关心过程细节"。这样 Agent 每次出的 PPT 和讲稿都自动对齐你的汇报标准。

6. **定期汇报用 Scheduled Tasks 提前准备**：如果你每个月都要做经营汇报，在 Cron Job 里创建一个"每月 25 号自动整理本月核心数据，生成 PPT 大纲草案"的定时任务。月底做 PPT 的时候，数据和框架已经帮你备好了，你只需要补充判断和亮点。
