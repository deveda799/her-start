# 推广投放的 CodeBanana 使用指南

> 推广的核心竞争力是"快"——比竞争对手更快找到高转化的素材和落地页。但传统流程里，做一版落地页要等开发一周，跑完 A/B 测试再迭代又是一周。CodeBanana 让你一天跑完三轮测试，把迭代周期从周压缩到小时。

---

## 场景一：1 天完成 3 轮落地页 A/B 测试

### 什么时候会用到

- 新品上线要投放广告，不确定哪种卖点更吸引人
- 正在跑的广告转化率不行，想快速测试几个新版本
- 老板说"把 ROI 提上去"，你需要找到最优落地页

### 怎么做

**上午 10:00——生成测试版本**

告诉 Agent 你要测什么：

> "做 3 个落地页测转化率，推广 AI 写作工具：
> - 版本 A 主打效率：标题'AI 写作，效率提升 10 倍'，红色按钮'立即体验'
> - 版本 B 主打质量：标题'AI 写作，媲美专业文案'，蓝色按钮'免费试用'
> - 版本 C 主打价格：标题'AI 写作，每天只需 3 元'，金色按钮'限时特惠'"

Agent 30 分钟生成 3 个独立页面，每个页面单独 **Deploy**，拿到 3 条永久链接。你复制链接设置广告投放，每个版本分配预算。链接不会过期，后面改了页面内容会自动更新——所以迭代的时候不需要换广告后台的链接。

**下午 14:00——看数据，快速迭代**

4 小时后数据出来：版本 B 转化率最高（2.5%）。你接着说：

> "基于版本 B 做 2 个优化版：B-1 把价格数字放大加粗，B-2 添加'已有 10 万人使用'"

20 分钟出新版本，继续投放。

**下午 17:00——锁定最优版本**

B-2 转化率 3.2%，全量投放。**一天之内完成了传统流程 4-6 周才能完成的优化。**

### 效果

| 指标 | 传统流程 | 用 CodeBanana |
|------|---------|--------------|
| 测试周期 | 2 周/轮 | 1 天/3 轮 |
| 迭代速度 | 1 周改一次 | 1 小时改一次 |
| 找到最优版本 | 4-6 周 | 1 天 |
| ROI 提升 | - | 更快找到高转化版本 |

## 场景二：推广日报自动化

### 什么时候会用到

- 每天要登录多个平台（百度、头条、微信）统计投放数据
- 老板每天问"昨天花了多少、转化多少"，你要手动汇总
- 某个渠道效果突然变差，发现的时候已经浪费了好几天预算

### 怎么做

一次设置，每天自动收到日报：

> "创建推广日报：每天早上 8 点生成昨日数据，包含总花费、各渠道点击量、转化量、ROI，ROI 低于 1.5 的标红预警"

从此每天早上 8 点，你自动收到：

```
推广日报 - 昨日数据

总花费：¥12,580 | 总转化：156 单 | 平均 ROI：2.1

渠道表现：
百度  ¥5,200  78单  ROI 2.5  ✅ 正常
头条  ¥4,800  62单  ROI 2.2  ✅ 正常
微信  ¥2,580  16单  ROI 1.1  🔴 预警

⚠️ 微信渠道 ROI 仅 1.1，建议检查素材或暂停投放
```

你 3 秒转发给老板（日报搞定），然后立刻去调整微信渠道（异常及时发现）。

如果你们团队用钉钉或飞书沟通，可以在 Agent Config 的 Channel 里绑定 IM 机器人。绑定后日报会直接推到群里，老板和同事都能第一时间看到，不用你手动转发。

### 效果

- 每天节省 30 分钟手动统计时间
- 异常渠道自动预警，不会等到月底复盘才发现问题
- 一个月节省 10 小时，可以多测试 2-3 个新渠道

## 给推广同学的小建议

1. **测试要有假设**：别只是随便做几个版本碰运气。每个版本测一个明确的假设（效率 vs 价格 vs 口碑），才能真正积累认知。

2. **保留历史数据**：让 Agent 把每次测试的数据和结论保存下来。时间长了你会发现规律——比如"促销风在节假日转化更好"。

3. **预警阈值别设太紧**：刚开始可以宽松一些（比如 ROI < 1.0 才预警），等数据稳定了再收紧，避免频繁误报。

4. **落地页和素材一起迭代**：让 Agent 同时做落地页和广告素材，保持视觉一致性，转化率会更好。

5. **日报用 Scheduled Tasks，异常检查用 Heartbeat**：每天的推广日报适合用 Cron Job 里的 Scheduled Tasks（每天早上 8 点生成）。但如果你想实时盯投放异常——比如"每小时检查一次各渠道 ROI，低于阈值就预警"——用 Heartbeat 更合适，写进 HEARTBEAT.md 设好间隔就行。

6. **接入 IM 让老板自动收到日报**：在 Agent Config → Channel 里绑定钉钉或飞书。日报和异常预警直接推到群里，老板不用问你"昨天数据怎么样"，你也不用每天手动转发。

7. **Basic Info 让 Agent 更懂投放**：在 User 里注明"推广投放岗位，关注 ROI、转化率、CPC 等核心指标"，Soul 里写"数据报告必须包含环比变化和异常标注，低于阈值的指标用红色标记"。这样 Agent 生成的报告天然就是你想要的格式。
