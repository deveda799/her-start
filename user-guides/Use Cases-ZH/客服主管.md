# 客服主管的 CodeBanana 使用指南

> 客服团队的痛点很集中：客诉记录散落在 Excel 和聊天记录里，跟进全靠人记，周报要手动统计。CodeBanana 能帮你把这些事情系统化——用对话录入代替填表，用定时任务代替手动检查，用自动统计代替周末加班做报表。

---

## 场景：20 分钟搭一套客诉管理系统

### 什么时候会用到

- 你的团队还在用 Excel 记客诉，经常忘记跟进
- 每周五要花 2 小时统计客诉数据做周报
- 你需要一个地方统一管理所有客诉，按严重程度跟进，不遗漏

### 第一步：创建系统（10 分钟）

告诉 Agent 你需要什么：

> "创建一个客诉管理系统，能记录客诉信息（客户名、问题类型、严重程度、跟进人、解决期限），每天提醒待跟进的，每周五自动出周报"

Agent 自动帮你搭好：
- 一个结构化的客诉记录文件
- 一个录入模板
- 三个核心操作：记录客诉、查看待跟进、生成周报

### 第二步：设定时提醒（5 分钟）

> "每天早上 9 点提醒今天要跟进的客诉，每周五下午 5 点生成周报"

Agent 创建两个定时任务，到点自动执行，不需要你做任何操作。

如果你的团队日常在飞书或钉钉沟通，还可以把 Agent 接入 IM——在 Agent Config 的 Channel 里绑定你们的飞书/钉钉机器人。绑定后，每天的客诉提醒和周报会直接推到群里，团队成员不用登录 CodeBanana 也能收到。

### 第三步：日常使用——说话就行

**录入客诉：**

> "记录客诉：京东李先生，产品质量问题，收到破损商品，特急，跟进人小王，3 天内解决"

Agent 自动格式化保存，严重程度标红，解决期限算好日期。

**每天早上 9 点自动收到提醒：**

```
今天有 3 个客诉需要跟进：

1. 京东李先生（特急🔴）- 破损商品 - 跟进人小王 - 还剩1天
2. 淘宝张女士（紧急🟡）- 物流延迟 - 跟进人小李 - 今天到期
3. 抖音王先生（一般）- 功能咨询 - 跟进人小刘 - 计划今天回访
```

你直接把提醒转发给对应同事，确保不遗漏。

**每周五下午 5 点自动收到周报：**

```
客诉周报 - 本周总览
- 本周客诉：15 单
- 已解决：12 单（80%）
- 超期：1 单 ⚠️

Top 3 高频问题：
1. 产品破损（7单）→ 建议加强包装
2. 物流延迟（5单）→ 建议更换快递商
3. 客服响应慢（2单）→ 建议增加人手
```

复制周报，直接用于周会汇报。

### 效果对比

| 功能 | 传统方式（Excel） | 用 CodeBanana |
|------|-----------------|---------------|
| 记录客诉 | 手动填表格，容易填错列 | 说一句话，自动格式化 |
| 跟进提醒 | 人工检查表格，容易遗漏 | 每天自动提醒，零遗漏 |
| 生成周报 | 周五花 2 小时统计 | 自动生成，5 秒完成 |
| 数据分析 | 手动筛选、做透视表 | 自动统计 + 自动预警 |

## 给客服主管的小建议

1. **先定义好问题类型**：让 Agent 知道你的客诉分类（产品质量、物流、服务态度等），它就能自动归类和统计。最好在 Basic Info 的 Agents 里写一份客诉处理 SOP——比如"特急客诉必须当天响应，普通客诉 3 天内跟进"，Agent 每次处理都会按这个标准执行。

2. **善用转发功能**：Agent 生成的每日提醒，直接转发给对应跟进人。不用你自己当人肉提醒器。

3. **接入 IM 降低门槛**：在 Agent Config → Channel 里绑定飞书/钉钉机器人。团队不一定每天打开 CodeBanana，但一定会看 IM。客诉提醒直接推到群里，谁都不会错过。

4. **超期自动升级**：你可以要求"超期 2 天的客诉自动标为特急"。用 Heartbeat 设一个"每 2 小时检查一次超期客诉"的任务，写进 HEARTBEAT.md——系统比人靠谱，它不会忘。

5. **用 Basic Info 设好客服标准**：在 Soul 里写上"语气专业但有同理心，回复客户时不使用推诿话术"。在 User 里注明"客服主管，关注客诉处理效率和客户满意度"。这样 Agent 在帮你起草回复或整理报告时，风格会自动对齐。
