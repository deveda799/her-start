# 全栈工程师的 CodeBanana 使用指南

> 工程师的日常不只是写新功能——大量时间花在搭项目、修 Bug、Code Review 这些重复性工作上。CodeBanana 不是替你写代码，而是像一个随时在线的高级同事：你说想法，它帮你落地；你贴报错，它帮你定位；你提交代码，它帮你把关。

---

## 场景一：从零搭一个完整项目

### 什么时候会用到

- 新项目启动，需要搭建前后端 + 数据库的完整骨架
- 你想用 Next.js + PostgreSQL 做个内部工具，但不想花半天配环境
- 老板说"做个 XX 管理系统"，你需要最快速度交付一个能跑的 MVP

### 怎么做

把你的需求描述清楚就行：

> "帮我搭一个任务管理系统，Next.js + PostgreSQL，要有任务的增删改查、状态流转、按负责人筛选"

Agent 会帮你完成整个技术栈的搭建：

1. **数据库设计**——自动创建 PostgreSQL 数据库，建表、加索引，数据结构帮你想好
2. **后端 API**——RESTful 接口，增删改查 + 筛选排序，错误处理都有
3. **前端页面**——完整的 UI，表格、表单、筛选器，响应式布局
4. **联调**——前后端连起来，数据能从数据库一路流到页面上

不满意的地方直接说：

> "表格加个批量删除功能，再加个导出 Excel"

Agent 改完你刷新页面就能看到效果。觉得差不多了？点 **Deploy**，拿到一条永久链接。这个链接不会过期，你后面改了代码它会自动更新——发给团队直接用，不用每次重新部署。

### 效果

从零搭一个全栈项目（数据库 + API + 前端 + 部署），传统流程至少 2-3 天。现在 30 分钟到 1 小时搞定，部署完团队立刻能用。

## 场景二：报错贴过来，帮你修

### 什么时候会用到

- 控制台一堆红字，看了半天不知道哪里出了问题
- 逻辑上有 Bug 但死活复现不出来
- 用户反馈线上白屏，你需要快速定位

### 怎么做

如果是你自己遇到的 Bug，可以直接在 **Private Agent** 里排查——不会影响 Team Agent 的上下文，也不用担心频繁的调试记录刷屏同事。

把报错信息直接贴过来：

> "线上用户反馈页面白屏了，这是控制台报错：
> `TypeError: Cannot read properties of undefined (reading 'map') at UserList (UserList.tsx:15)`"

Agent 会立刻做三件事：

1. **定位问题**——错误在哪个文件哪一行，变量为什么是 undefined
2. **给出修复方案**——不只是修这一行，还会补上加载状态、错误处理、空数据展示
3. **直接改代码**——你不用手动改，Agent 帮你改好

如果还有连带问题，继续说：

> "改完了，但列表加载特别慢，要 5 秒"

Agent 会继续排查——查 API 响应时间、检查 SQL 查询有没有分页、有没有加索引，一路追到根因。

### 效果

以前排查一个 Bug，尤其是不熟悉的代码库里的 Bug，可能要半天。现在贴报错 → 定位原因 → 给修复方案 → 改好代码，5 分钟搞定。

## 场景三：代码审查，帮你把关

### 什么时候会用到

- 写完代码准备提交，想自己先过一遍
- 同事的代码 Review 排不上，但又不想直接合并
- 涉及支付、权限等敏感逻辑，想多一双眼睛看看

### 怎么做

把代码丢给 Agent：

> "帮我 Review 一下这个支付接口的代码"

Agent 会像一个严格的 Senior Engineer 一样逐项审查，报告分级输出：

- **🔴 严重问题**（必须修）：SQL 注入、金额未校验等安全漏洞
- **🟡 改进建议**（建议修）：错误处理不完整、缺少日志、函数太长
- **📊 总结**：代码质量评分 + 优先处理顺序

不只是找问题，还直接给修复代码。觉得严重问题要改？

> "帮我把严重问题都改掉"

Agent 直接改好，你只需要 Review 改动就行。

### 效果

以前找同事 Review 要等排期，自己看容易有盲区。现在随时能审查，安全漏洞、性能问题、可读性问题一个不漏。对于支付、权限这类高风险代码，多一层 AI 审查，心里更踏实。

## 场景四：用 Basic Info 打造你的专属编程搭档

### 什么时候会用到

- Agent 写的代码风格跟你的项目不一致（比如你用 4 空格缩进它用 2 空格）
- 每次都要提醒"用 TypeScript 不要用 JavaScript"
- 涉及敏感操作时希望 Agent 先问你，而不是直接执行

### 怎么做

进入项目，点右上角 **「···」** → **「Basic Info」**，调整这几个 tab：

**Agents（写入你的代码工作流 SOP）：**

```
1. 写代码前先读 MEMORY.md，了解项目技术栈和历史决策
2. 优先保证代码可读性，函数不超过 50 行，文件不超过 300 行
3. 每次修改后必须运行 linter 和单测
4. 遇到架构级变更，先列方案让我确认再动手
```

**Soul（设定安全红线）：**

```
1. 务实、高效的高级全栈工程师，推崇 Keep It Simple
2. 绝对红线：
   - 不执行 rm -rf / 或任何不可逆的删除命令
   - 不在代码中明文写入 API Keys 或密码，必须用环境变量
   - 不确定安全性时必须先问我
3. 代码风格：4 空格缩进，必须使用 TypeScript，禁止 any 类型
```

**User（告诉 Agent 你的技术背景）：**

```
高级全栈工程师，主要技术栈：Next.js + TypeScript + PostgreSQL
部署环境：Vercel + Neon
不需要解释基础概念，直接给方案和代码
```

**Memory** 不需要你手动设——Agent 会在工作过程中自己积累。比如你纠正过一次"不要用 var"，它就会记住，以后不会再犯。

### 效果

相当于给 Agent 装了一套你的代码规范。以后每次写代码都自动遵守——缩进风格、命名规范、安全红线，不用每次提醒。

## 给工程师的小建议

1. **调试用 Private Agent，协作用 Team Agent**：自己排查 Bug 在 Private Agent 里搞定，不会刷屏同事。需要团队一起看的代码变更，再到 Team Agent 里讨论。

2. **用 Agent 处理你不想做但必须做的事**：写单测、补注释、加日志、格式化代码——这些事交给 Agent，你专注核心逻辑。

3. **Code Review 养成习惯**：每次提交前让 Agent 过一遍，相当于免费的 Senior Review。尤其是涉及安全和性能的代码。

4. **重复检查交给 Cron Job**：Heartbeat 适合"每隔一段时间跑一遍"的任务——比如"每 30 分钟跑一遍测试用例"、"每小时检查一次线上接口"。设好后 Agent 自动执行，有问题主动通知你。精确时间的任务（"每天早上 9 点跑全量测试"）用 Scheduled Tasks。

5. **Basic Info 就是你的编码规范文档**：花 10 分钟把代码风格、安全红线、工作流写进 Agents 和 Soul 里，比口头约定靠谱。新成员加入项目后，Agent 也会按同一套规范工作。
