# 法务的 CodeBanana 使用指南

> 法务的工作量很大一部分花在"看文字"上——起草合同逐条写、审合同逐条读、对比版本逐字比。这些工作的共同特点是：需要极高的注意力，但模式是可重复的。CodeBanana 不能替你做法律判断，但它能帮你大幅加速文字处理环节，让你把精力放在真正需要专业判断的地方。

---

## 场景一：15 分钟起草一份合同

### 什么时候会用到

- 业务部门说"帮我出份合同"，你要从零开始写
- 有模板但需要针对特定客户和场景做定制
- 需要快速出一份框架，后续再精修

### 怎么做

把业务背景告诉 Agent：

> "帮我起草一份 SaaS 服务协议，我们是服务提供方，客户是一家银行，年费 50 万，服务期 1 年，SLA 99.9%"

Agent 15 分钟交付完整合同初稿：定义与解释、服务内容与范围、费用与支付、SLA 承诺及赔偿、数据安全与保密……该有的章节都有，而且会自动考虑行业特点（比如银行客户会加金融合规条款和数据处理协议）。

需要调整？

> "加一条竞业限制条款，SLA 赔偿比例改成月费 10%"

30 秒更新。你在这个初稿基础上做专业审核和精修，效率远比从零写高。

### 效果

以前起草合同——找模板、参考案例、逐条撰写——至少 2-3 天。现在 15 分钟出完整初稿，省下来的时间用于法律条款的推敲和风险判断。

## 场景二：10 分钟审完一份合同的风险

### 什么时候会用到

- 收到对方发来的合同，需要快速识别不利条款
- 签署前做全面风险评估
- 业务催着说"合同今天能签吗"，你需要尽快给结论

### 怎么做

合同内容通常比较敏感，建议在 **Private Agent** 里操作——只有你自己能看到对话内容，不会出现在团队的 Team Agent 里。

把合同文件发给 Agent：

> "帮我审一下客户发来的这份采购合同"

Agent 10 分钟出一份风险审查报告，按严重程度分级：

**🔴 高风险（必须改）：**
- 无限连带责任条款——赔偿无上限
- 单方解约无补偿——对方可随时终止

**🟡 中风险（建议改）：**
- 知识产权归属模糊——共有权容易起纠纷
- 付款条件苛刻
- 验收标准未明确

每个风险点都标注了：原文位置、风险分析、修改建议。

> "帮我把修改意见直接标注在合同里"

Agent 在对应位置插入批注，输出带标记的版本，你直接发给对方谈判。

### 效果

以前审一份合同逐条读要 1-2 小时。现在 10 分钟出分级风险报告，高中低一目了然。你可以把注意力集中在高风险条款的法律判断上，而不是花在逐字阅读上。

## 场景三：3 分钟对比两版合同差异

### 什么时候会用到

- 对方发回了修改版，你需要逐条确认改了哪些
- 谈判了好几轮，要确认己方的修改意见是否被采纳
- 合同经过多人修改，需要和原版做全量对比

### 怎么做

把两个版本的合同都发给 Agent：

> "对方发回了修改版，帮我和我们上一版对比一下"

Agent 3 分钟出完整对比报告：

- **重要变更**：付款周期从 15 天被改成 45 天、赔偿上限被降低、终止条件被改为仅甲方可终止
- **次要变更**：新增排除条款、修改验收流程
- **已采纳我方意见**：知识产权、保密期限、争议解决——3 处已按我方要求修改

每处差异都标注了原文对比和影响分析。

> "不同意的标红，同意的标绿，导出 Word"

Agent 标注颜色，输出带批注的 Word 文档。你拿着这份对比去谈判，对方改了什么一清二楚。

### 效果

以前两版合同逐字对比要 1 小时，而且总怕漏掉细微改动。现在 3 分钟出完整对比，不会遗漏任何一处差异。

## 给法务的小建议

1. **Agent 出的是初稿，不是终稿**：合同涉及法律风险，Agent 帮你加速文字处理，但最终的法律判断必须由你来做。把它当成一个极快的助理，而不是一个律师。

2. **敏感合同用 Private Agent**：涉及未公开的交易条款、竞业协议、并购文件等，在 Private Agent 里处理。只有你能看到内容，团队其他人不可见。

3. **建立自己的模板库**：每次起草或审查后，把定稿保存在项目里。下次类似场景直接告诉 Agent"参考上次的 SaaS 协议模板"，效率更高。

4. **版本对比要养成习惯**：每轮谈判后都用 Agent 做一次对比，确保没有遗漏的改动。这个 3 分钟的投入能帮你避免很多后续纠纷。

5. **用 Basic Info 设好法务边界**：在 Soul 里明确写上"你是法务助理，只负责文字处理和风险提示，不做最终法律判断。涉及模棱两可的条款，必须标注为'需人工审核'而不是自行定性"。在 User 里写"法务岗位，熟悉合同法和公司法，输出格式参照法律文书标准"。这样 Agent 既高效又不会越界。

6. **合同到期提醒用 Scheduled Tasks**：在 Cron Job 里为重要合同设"一次性"定时任务——比如"合同到期前 30 天提醒我启动续约流程"。多份合同就创建多个任务，Agent 到点自动提醒，不怕漏掉续约窗口。
