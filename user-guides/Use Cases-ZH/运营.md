# 运营的 CodeBanana 使用指南

> 运营是最需要"看数据"的岗位之一——搭看板、写周报、查异常，每一件都要花时间。但这些工作的核心价值不在于统计数字本身，而在于从数字里发现问题、做出决策。CodeBanana 帮你把统计的活干了，让你专注于"看懂数据"和"做对决策"。

---

## 场景一：30 分钟搭一个数据看板

### 什么时候会用到

- 你需要一个日常监控运营核心指标的面板
- 找开发做看板要排期 2-3 周，等不了
- 想把散落在各个表格里的数据汇总到一个页面

### 怎么做

告诉 Agent 你要看什么数据：

> "帮我搭一个电商运营数据看板，给运营团队日常用"

Agent 自动设计方案并生成可交互的看板页面：

- **顶部数字卡片**——今日 GMV、订单数、转化率、客单价，一眼看大盘
- **中间趋势图**——近 30 天 GMV 趋势（含同比）、品类销售占比、转化漏斗
- **底部明细表**——商品销量 Top10、各渠道流量来源
- **自动预警规则**——转化率 < 2% 报红、日 GMV < 目标 80% 亮黄、退款率 > 5% 报红

需要加指标？

> "再加一个客服响应时长"

30 秒更新。看板搭好后点 **Deploy**，拿到一条永久链接——发给运营团队每个人，大家打开书签就能看实时数据。你后面加了新指标或改了布局，链接会自动更新，不用重新发。

### 效果

以前搭看板要等开发排期 2-3 周。现在 30 分钟自己搭好，随时加指标随时改布局。

## 场景二：10 分钟出一份数据周报

### 什么时候会用到

- 每周要给领导汇报运营数据，需要有图表、有分析、有结论
- 手头一堆数据但没时间整理成报告
- 想把"做报告"的时间省下来用在"做运营"上

### 怎么做

不想每周手动操作？设个定时任务：

> "每周五下午 4 点自动生成本周运营周报"

从此每周五你打开就有现成的周报等着。如果团队用飞书或钉钉，还可以把 Agent 接入 IM——周报生成后直接推到群里，领导打开群就能看到。

当然，第一次你需要告诉 Agent 数据来源和格式偏好：

> "用这份数据帮我出本周运营周报"

Agent 10 分钟交付完整周报：

- **概览**：总访问量、转化订单、GMV、客诉——核心数字 + 环比变化
- **趋势图**：自动生成日活、转化、GMV 的 7 天趋势折线图
- **亮点**：周三大促带来访问量峰值，新用户占比提升
- **问题**：周末转化率偏低，退款率上升
- **下周建议**：周末增加限时活动、排查退款原因、继续加大拉新

给老板看觉得太详细？

> "改成给 CEO 看的版本，精简一些"

Agent 精简为 3 张核心图表 + 5 行结论。

### 效果

以前每周花 3 小时做 Excel 图表 + 写 Word 报告。现在 10 分钟出完整周报，还有图表和建议。

## 场景三：指标异常？5 分钟帮你排查原因

### 什么时候会用到

- 转化率突然掉了，不知道是什么原因
- 某个指标大幅波动，需要快速做归因分析
- 老板问"怎么回事"，你需要尽快给出答案

### 怎么做

> "今天转化率从 3.2% 掉到 1.5% 了，帮我查一下原因"

Agent 自动做层层下钻分析：

**第 1 层——分渠道看：** 自然搜索正常、直接访问正常，付费投放异常 → 问题锁定在付费投放

**第 2 层——分投放计划看：** 百度品牌词正常、朋友圈正常，头条信息流异常 → 问题进一步锁定

**第 3 层——看头条信息流明细：** 今天 10:00 切换了新素材，新落地页加载时间从 2 秒变成 8 秒，跳出率飙升到 85%

**结论：** 头条信息流今天换的新落地页加载过慢导致用户大量跳出。

**建议：** 立即回退旧落地页，排查新页面性能问题后再上线。

从发现异常到定位根因，5 分钟。

### 效果

以前指标异常要运营 + 技术 + 投放三方拉会排查，半天到一天。现在 5 分钟层层下钻直接定位根因，快速止损。

## 给运营的小建议

1. **看板部署后加到书签**：Deploy 生成的是永久链接，团队每个人加到浏览器书签，每天打开就能看数据。你改了看板指标，链接自动更新，不用通知大家。

2. **周报用 Scheduled Tasks，异常检查用 Heartbeat**：在 Cron Job 里，"每周五下午 4 点生成周报"适合用 Scheduled Tasks（精确时间触发）。而"每 30 分钟检查一次核心指标有没有异常"适合用 Heartbeat——把检查项写进 HEARTBEAT.md，设好间隔，Agent 自动巡检。两者配合使用，日常报告和实时预警全覆盖。

3. **预警推到 IM 群**：在 Agent Config → Channel 里接入飞书或钉钉。指标异常时 Agent 直接在群里通知。比你自己发消息快，也不会因为开会忙而漏掉预警。

4. **异常排查要快、止损要更快**：发现问题后先止损（回退、暂停），再慢慢排查根因。别在排查上花太多时间而让问题持续影响数据。

5. **用 Basic Info 让 Agent 自带运营视角**：在 User 里写"运营岗位，日常关注 GMV、转化率、留存率、客单价"，Soul 里写"数据报告必须包含环比变化和趋势判断，异常指标用红色标注，每份报告结尾给出可执行的下一步建议"。这样 Agent 输出的所有分析都自动带上运营思维，不用每次都强调"要有结论和建议"。
