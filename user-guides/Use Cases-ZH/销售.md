# 销售的 CodeBanana 使用指南

> 销售最怕两件事：忘了跟进客户、开会被问竞品答不上来。CodeBanana 能帮你把客户管理系统化——说一句话就能录入客户信息，每周自动提醒该跟谁，周报不用自己整理。竞品动态也帮你自动盯着，周一开会直接用。

---

## 场景一：10 分钟搞定客户跟进管理

### 什么时候会用到

- 客户信息散落在 Excel、微信、笔记本里，经常找不到
- 跟进提醒全靠自己记，偶尔会忘
- 周报要花 1 小时回忆本周都跟了谁

### 第一步：创建系统（3 分钟）

> "帮我创建一个客户跟进管理系统"

Agent 自动搭好：客户清单文件、跟进记录文件，支持"添加客户"、"记录跟进"、"生成周报"三个操作。

### 第二步：设自动提醒（2 分钟）

> "每周一早上 9 点提醒我本周要跟进的客户"

从此每周一早上你会收到一份清单——谁该打电话、谁该发方案、谁该回访，不用自己翻日历。

如果你平时主要用企业微信或钉钉，可以把 Agent 接入 IM（在 Agent Config → Channel 里设置）。接入后，每周一的客户提醒会直接推到你的企微/钉钉，不用打开 CodeBanana 也能收到。

### 第三步：日常使用——说话就行

**录入客户：**

> "添加客户：万达张总，负责采购，上次沟通了产品升级需求，下次跟进 3 月 25 日"

Agent 自动保存，格式化好。

**记录跟进：**

> "今天联系了万达张总，确认预算 200 万，要求月底前给方案"

Agent 追加记录到跟进日志，自动标注日期和下一步。

**生成周报：**

> "生成本周销售周报"

Agent 自动汇总：本周拜访了几家、新增几条线索、哪些确认了意向、下周计划是什么。你复制发给领导就行。

### 效果

| 维度 | 传统方式 | 用 CodeBanana |
|------|---------|--------------|
| 客户信息 | 散落各处，经常找不到 | 一个地方统一管理 |
| 跟进提醒 | 手动记日历，容易忘 | 每周一自动提醒 |
| 周报生成 | 花 1 小时回忆整理 | 5 秒自动生成 |

## 场景二：竞品自动监控

### 什么时候会用到

- 每周开会要汇报竞品情况，但没空天天盯
- 想知道竞品最近有没有降价、发新版、签大客户
- 客户问"你们比 XX 好在哪里"，你需要有底气地回答

### 怎么做

一次设置：

> "创建竞品监控任务：每周五下午 5 点，搜索竞品 A、B、C 的最新动态，整理成对比表格"

从此每周五下班前自动收到：

```
竞品周报

| 竞品 | 本周动态 | 价格变化 | 关键信息 |
|------|---------|---------|---------|
| 竞品A | 发布新版本 | 降价 10% | 增加 AI 功能 |
| 竞品B | 无更新 | 无变化 | - |
| 竞品C | 新签大客户 | 无变化 | 进入金融行业 |
```

周一开会直接用，不用临时搜索。客户问竞品对比，你心里有数。

### 效果

- 不用每周手动查竞品官网
- 关键信息自动整理好，省去总结时间
- 周五收到，周一开会直接用

## 给销售的小建议

1. **每次拜访后立刻记录**：打完电话、见完客户，花 10 秒钟告诉 Agent 关键信息。信息趁热记最准确，拖到周末就忘了一半。

2. **善用周一提醒**：每周一看完提醒后，花 5 分钟规划本周跟进顺序。有计划的跟进比凭记忆跟进靠谱得多。Cron Job 里的 Scheduled Tasks 支持 Weekly 类型，"每周一早上 9 点"这种精确时间触发非常适合。

3. **竞品信息随时更新**：客户告诉你竞品的消息？立刻让 Agent 记下来。这些一线情报比网上搜索的更有价值。

4. **周报不只是给领导看的**：每周回顾一下自己的跟进记录，看看哪些客户好几周没动静了。有时候一个被忽略的客户才是最大的机会。

5. **接入企微/钉钉，随时随地记录**：在 Agent Config → Channel 里把 Agent 接入你常用的 IM。在外面见完客户，直接在企微/钉钉群里 @ 机器人说"记录：今天见了 XX 客户，确认预算 XX"——不用打开电脑，不用登录 CodeBanana，走在路上就能把跟进信息录进去。

6. **用 Basic Info 让 Agent 懂销售语言**：在 User 里注明"销售岗位，关注客户跟进、商机管理、竞品动态"，Soul 里写"输出风格简洁直接，用表格呈现客户清单，周报必须包含本周新增线索数、跟进数、意向确认数"。设好之后 Agent 的输出天然就是你想要的销售报告格式。
