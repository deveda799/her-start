# HR（招聘）的 CodeBanana 使用指南

> 招聘最耗时的不是面试本身，而是面试前后的准备工作——写 JD 要参考一堆模板拼凑，筛简历要逐份看，出面试题要临时想。这些都是有明确标准、可以结构化处理的工作。CodeBanana 帮你把这些环节从"小时级"压缩到"分钟级"，让你把精力放在和候选人的沟通判断上。

---

## 场景一：5 分钟写好一份岗位 JD

### 什么时候会用到

- 业务部门说"帮我招个人"，你需要快速写出 JD 发布到招聘平台
- 现有 JD 投了一周没几个人投，需要优化吸引力
- 同一个岗位要发多个平台，每个平台风格不一样

### 怎么做

把岗位的核心信息告诉 Agent：

> "帮我写一份高级前端工程师的 JD，我们做 AI 产品，团队 10 人，技术栈 React + TypeScript"

Agent 5 分钟交付完整 JD：

- **为什么加入我们**——不是公司简介的复制粘贴，而是真正吸引候选人的理由
- **你将负责**——具体的职责描述，不是"完成领导安排的其他工作"
- **我们希望你**——必备条件和加分项分开列，候选人一目了然
- **我们提供**——薪资范围、福利亮点

需要发多个平台？

> "再出一版简短的，用于 Boss 直聘"

Agent 精简为 300 字版本，突出薪资和亮点，适配平台风格。

### 效果

以前写 JD——参考模板、拼凑内容、反复修改——半天。现在 5 分钟出专业 JD，还能一键适配不同平台。

## 场景二：50 份简历 10 分钟筛完

### 什么时候会用到

- 岗位发出去收到大量简历，逐份看太耗时
- 需要快速筛选出最合适的候选人安排面试
- 需要对比几个候选人，不知道该先面谁

### 怎么做

简历涉及候选人个人信息，建议在 **Private Agent** 里处理——只有你可见，不会出现在团队对话里。

把简历和 JD 一起发给 Agent：

> "这 50 份简历帮我筛一下，岗位要求是这个 JD"

Agent 10 分钟出一份结构化的筛选报告：

**推荐面试（Top 5）：**
- 🥇 张三 92 分——React 7 年、有 AI 产品经验、开源项目 2.5k star → 强烈推荐
- 🥈 李四 85 分——大厂背景、性能优化经验丰富、无 AI 经验但可培养 → 推荐
- 🥉 王五 78 分……

**淘汰名单（附原因）：**
- 候选人 A：经验不足（仅 2 年）
- 候选人 B：技术栈不匹配（Vue 为主）
- 候选人 C：薪资预期超出范围

想深入对比？

> "张三和李四详细对比一下"

Agent 逐项对比技术栈、项目经验、软实力，输出对比表格。

### 效果

以前 50 份简历看一整天，还可能因为疲劳漏掉好候选人。现在 10 分钟出筛选结果，每人有打分和推荐理由，你只需要 review 前 5 名就行。

## 场景三：5 分钟出一套面试题

### 什么时候会用到

- 明天面试，需要准备针对性的问题
- 不同候选人背景不同，不想用同一套题
- 需要包含技术题、场景题、软实力的完整面试方案

### 怎么做

把候选人简历和岗位要求发给 Agent：

> "明天面试张三，高级前端岗位，他简历上 React 和性能优化经验比较强"

Agent 5 分钟出完整面试方案（60 分钟）：

- **技术基础（15 分钟）**——考底层原理理解深度，带考察点和期望答案
- **项目深挖（20 分钟）**——针对简历上的具体项目追问细节，考实战能力
- **场景题（15 分钟）**——假设题考方案设计能力 + 团队协作能力
- **候选人提问（10 分钟）**——预留时间
- **评分标准**——技术深度、项目经验、设计能力、沟通协作，各占多少权重

每道题都标注了**考察点**和**追问方向**，让面试官知道从哪里深挖。

> "再加两道 AI 相关的题，毕竟我们做 AI 产品"

30 秒补充完成。

### 效果

以前面试前花 1 小时想题目，还可能临时想不到好问题。现在 5 分钟出一整套针对性面试题，带评分标准，面试质量更高。

## 给 HR 的小建议

1. **简历筛选用 Private Agent**：候选人信息比较敏感，在 Private Agent 里处理更合适。筛选结果确认后，再把推荐名单分享给用人部门。

2. **面试安排用 Scheduled Tasks**：在 Cron Job 里创建一次性定时任务（选 Once）——比如"明天下午 2 点面试前 1 小时提醒我，附上候选人简历摘要和面试题"。这样你不用面试前临时翻资料。日常周期性的任务（比如"每周一汇总本周面试安排"）用 Weekly 类型。

3. **接入 IM 同步面试进展**：在 Agent Config → Channel 里把 Agent 接入飞书或钉钉。面试安排、候选人评估结果可以直接推到招聘群。用人部门不用一个个问你"面试安排到哪了"。

4. **面试题库积累起来**：每次出的面试题保存下来，按岗位分类。下次招类似岗位，让 Agent 在已有题库基础上定制，越用越好用。

5. **用 Basic Info 设好保密规范**：在 Soul 里写上"候选人信息严格保密，不在团队对话中泄露个人信息"、"薪资数据不允许出现在 Discussion 中"。在 User 里注明"HR 岗位，输出内容面向内部招聘流程"。这样 Agent 在处理候选人相关任务时，会自动遵守保密规则。
