# 市场部的 CodeBanana 使用指南

> 市场部最缺的不是创意，是执行速度。一个活动页面从构思到上线要等设计和开发排期两周，一套营销素材要在文案、设计、排版之间来回沟通。CodeBanana 让你跳过这些等待——自己就能做页面、出素材、盯竞品。

---

## 场景一：1 小时搞定活动落地页

### 什么时候会用到

- 618、双 11 要做活动页面，不想等设计和开发排期
- 老板临时要一个产品展示页，下午就要
- 需要快速上线一个推广页面投放广告

### 怎么做

把你的活动信息告诉 Agent：

> "做一个 618 活动页面：标题'618 狂欢，全场 5 折'，顶部要科技风大图，红色金色配色，中间展示 3 个产品（名字、原价、现价、购买按钮），底部放倒计时，适配手机"

Agent 15 分钟生成完整页面，响应式布局、倒计时代码都包含。打开预览链接直接看效果。

不满意的地方当场改：

> "标题字号再大一些，按钮改成圆角"

改完刷新就能看到。确认没问题后：

> "加上百度统计代码，追踪页面访问量和按钮点击"

统计代码插好，测试确认上报正常。最后点 **Deploy**，拿到一条永久链接。这个链接不会过期——发微信群、发朋友圈、投广告后台都行。后面你改了页面内容（比如临时加个优惠），链接会自动更新，不用重新发地址。

### 效果

| 环节 | 传统流程 | 用 CodeBanana |
|------|---------|--------------|
| 设计视觉稿 | 2 天 | 0（直接生成） |
| 等开发排期 | 3-5 天 | 0 |
| 开发实现 | 3 天 | 15 分钟 |
| 测试修改 | 1-2 天 | 15 分钟 |
| 部署上线 | 半天 | 15 分钟 |
| **总耗时** | **10-14 天** | **1 小时** |

## 场景二：10 分钟批量生成营销素材

### 什么时候会用到

- 要推一个新产品，需要多种风格的文案选优
- 需要文案 + 配图 + PPT 一站式产出
- 同一个活动要在多个渠道投放不同风格的素材

### 怎么做

一句话就能出全套：

> "给产品 X 生成推广素材"

Agent 自动产出：
1. **5 种风格文案**——科技风、情怀风、促销风、口碑风、场景风
2. **每条文案配图**——5 张风格匹配的配图
3. **自动整理成 PPT**——标题页 + 每页一种风格（图 + 文案）+ 对比页

你打开 PPT，挑最喜欢的版本，直接投放。

需要微调？

> "把情怀风文案改成'十五年专注'，重新配图"

30 秒更新完成。

### 效果

以前做一套营销素材，文案、设计、排版来回沟通 3-5 天。现在一句话出 5 套方案，不满意 30 秒改版。

## 场景三：竞品监控自动化

### 什么时候会用到

- 每周开会要汇报竞品动态，但没时间天天查
- 想知道竞品最近有什么新动作（发版、降价、签大客户）
- 需要定期的竞品对比报告

### 怎么做

一次设置，永久受益：

> "创建竞品监控任务：每周五下午 5 点，搜索竞品 A、B、C 的最新动态，整理成对比表格"

Agent 创建定时任务。从此每周五下班前，你自动收到：

```
竞品周报 - 本周动态

| 竞品 | 本周动态 | 链接 | 关键信息 |
|------|---------|------|---------|
| 竞品A | 发布 3.0 版本 | [链接] | 新增 AI 功能，降价 10% |
| 竞品B | 无重大更新 | - | - |
| 竞品C | 签约某银行 | [链接] | 进入金融行业 |

本周重点：
- 竞品A 降价可能影响 Q2 销售，建议关注
- 竞品C 进入金融行业，与我们目标市场重叠
```

周一开会直接用，不用临时抱佛脚。

## 给市场部的小建议

1. **活动页面先出 MVP**：别一开始就想做得完美。先让 Agent 出一个基础版，看看感觉对不对，再一步步调细节。反正部署链接是自动更新的，改完即生效。

2. **素材库积累起来**：每次生成的文案和配图都保存好，慢慢就有了自己的素材库。下次做类似活动可以在已有素材基础上改。

3. **竞品监控用 Scheduled Tasks**：在 Cron Job 里创建定时任务，比如"每周五下午 5 点搜索竞品动态"。如果你还想高频盯——比如电商大促期间每 2 小时查一次竞品价格——可以用 Heartbeat，把检查项写进 HEARTBEAT.md，设好间隔就行。

4. **部署链接放心发**：Deploy 生成的是永久链接，改了页面会自动更新。你可以放心把链接贴到各种地方——朋友圈、广告后台、合作方文档——不用担心失效。

5. **Basic Info 让 Agent 自带营销思维**：在 Basic Info 的 User 里写"市场部成员，输出内容面向客户"，Soul 里写"文案风格：有感染力、重点突出、善用数据对比"。这样 Agent 生成的文案和报告天然带营销味道，不用每次都强调"别写得太技术"。

6. **把结果推到 IM 群**：在 Agent Config → Channel 里接入飞书或钉钉。竞品周报、活动数据这些定时任务的结果会直接推到群里，老板和同事不用登录 CodeBanana 也能第一时间看到。
