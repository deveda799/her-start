# 财务的 CodeBanana 使用指南

> 财务工作对准确性要求极高，但大量时间花在重复性的核对和统计上——对账要逐行比、报表要逐项查、分析报告要手动做图表再写结论。CodeBanana 能帮你加速这些机械性环节，让你把注意力放在发现异常和做出判断上。

---

## 场景一：10 分钟完成对账

### 什么时候会用到

- 月底要核对银行流水和系统记录
- 两份数据对不上，需要找出差异在哪
- 需要生成对账报告或银行余额调节表

### 怎么做

财务数据通常比较敏感，建议在 **Private Agent** 里操作——对话内容只有你可见，不会出现在团队的 Team Agent 里。

把两份数据发给 Agent：

> "帮我对一下这两份数据，银行流水和我们系统的记录"

Agent 自动做三件事：
1. 解析两份数据，统一格式
2. 按金额 + 日期自动匹配
3. 找出差异项，分析可能原因

10 分钟后输出对账报告：

```
对账结果：
- 银行流水 156 笔，系统记录 152 笔
- 匹配成功 149 笔
- 差异 7 笔，差额 ¥7,500

差异明细：
1. 3/15 银行 ¥5,000，系统无 → 疑似遗漏，请补录
2. 3/18 银行 ¥12,000 vs 系统 ¥12,500 → 金额不一致，请核实合同
3. 3/22 系统 ¥2,000，银行无 → 在途款项，预计 3/25 到账
```

每笔差异都给出原因分析和建议操作。

> "帮我生成调账分录"

Agent 自动生成对应的调账分录。

### 效果

以前手动对账一个下午，眼花了还可能漏掉。现在 10 分钟出结果，差异原因和下一步操作都帮你分析好。

## 场景二：5 分钟校验一份报表

### 什么时候会用到

- 报表做完了，提交前需要做最终检查
- 不确定有没有公式错误或逻辑异常
- 同比环比数据变化太大，想确认是真实变动还是录入错误

### 怎么做

把报表发给 Agent：

> "帮我检查一下这份季度利润表"

Agent 自动校验并分三类输出：

**✅ 通过项：**
- 借贷平衡：通过
- 合计公式：15 个全部正确

**⚠️ 异常项：**
- 管理费用-差旅费本季度 ¥89,000，同比增长 320%，远超历史波动
- 营业收入第 2 行和第 5 行合计不等于汇总行，差额 ¥1,200
- 其他应付款期末余额为负数，疑似借贷方向录反

**📊 趋势提醒：**
- 毛利率从 42% 降到 35%，关注成本变化
- 应收账款周转天数增加到 45 天

你确认差旅费是合理的、让 Agent 帮你查漏录的那笔，一来一回 5 分钟搞定。

### 效果

以前自己逐项核对 1-2 小时，还怕遗漏。现在 5 分钟出完整校验报告，异常值一目了然。

## 场景三：15 分钟出一份经营分析报告

### 什么时候会用到

- 给老板准备经营分析汇报材料
- 有一堆财务数据但没时间整理成"能讲的"报告
- 需要同比环比分析 + 趋势判断 + 经营建议

### 怎么做

把数据发给 Agent：

> "用这份数据帮我做个 Q1 经营分析报告，给老板看的"

Agent 15 分钟交付：

- **核心指标一览**：营收、净利润、毛利率——实际 vs 目标 vs 同比
- **收入分析**：哪个产品涨了、哪个跌了、新客户贡献占比
- **成本预警**：原材料上涨 18%、营销费用超支 20%
- **现金流**：经营性现金流健康，但应收账款周期延长
- **建议**：调整定价策略、控制营销费用、加强应收催收

同时生成配套图表——收入趋势折线图、成本结构饼图、现金流柱状图。

> "再加上和去年 Q1 的对比分析"

30 秒补充完成。

### 效果

以前做经营分析要 1-2 天——Excel 做图表、Word 写报告来回切换。现在 15 分钟出报告 + 图表，数据解读和建议都帮你写好。你只需要 review 一遍确认数据准确。

## 给财务的小建议

1. **敏感数据用 Private Agent**：对账、报表、经营分析涉及公司核心财务数据，在 Private Agent 里处理更安心。只有你能看到对话内容。

2. **建立对账模板**：告诉 Agent 你的对账规则（比如"金额一致且日期在 3 天内算匹配"），它会按你的标准执行，保持一致性。最好把规则写进 Basic Info 的 Agents 里，形成固定的对账 SOP——比如"对账步骤：1.解析文件 2.统一格式 3.按金额+日期匹配 4.差异项输出原因分析"。

3. **月度报告用 Scheduled Tasks**：在 Cron Job 里创建"每月 1 号自动生成上月经营分析报告"的定时任务（选 Monthly 类型）。你到了月初打开就有现成的报告框架，在上面补充判断和批注就行，不用从零做起。

4. **数据准确性由你把关**：Agent 帮你加速处理，但最终数字必须由你确认。尤其是对外报告，一定要 review。在 Basic Info 的 Soul 里写上"数据报告必须标注数据来源和统计口径，任何手动调整项必须用黄色高亮标注"——这样 Agent 输出的报告自带审计线索。

5. **异常指标用 Heartbeat 盯着**：如果你需要持续监控某些指标（比如"现金流低于安全线"、"应收账款超期"），把检查项写进 HEARTBEAT.md，设好间隔。Heartbeat 会定期自动检查，发现异常立刻通知你，比你手动查 Excel 靠谱得多。
