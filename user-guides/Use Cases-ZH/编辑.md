# 编辑的 CodeBanana 使用指南

> 编辑的时间永远不够用——选题要想、初稿要写、校对要细、还要适配不同平台的风格。CodeBanana 能帮你压缩"从 0 到初稿"的时间，让你把精力花在真正体现编辑功底的地方：打磨文字、把控调性、做好内容判断。

---

## 场景一：15 分钟出一篇 2000 字初稿

### 什么时候会用到

- 有选题但没时间从零写起，需要一个初稿作为起点
- 手头有素材和数据，需要快速整合成完整文章
- 被催稿了，需要先出一版再慢慢改

### 怎么做

把选题和要求告诉 Agent：

> "帮我写一篇公众号文章，主题是'中小企业如何用 AI 降本增效'，目标 2000 字，轻松活泼的风格"

Agent 交付的不只是正文——还包括：
- 1 个主标题 + 2 个备选标题
- 100 字摘要
- 完整正文，分段清晰，有小标题
- 自动适配公众号风格：短段落、重点加粗、口语化表达

觉得哪里不对？

> "语气再正式一些，第二部分加几个数据支撑"

30 秒调整完成。你在这个初稿基础上打磨、加入自己的观点和素材，比从零写快得多。

### 效果

以前从零写一篇 2000 字至少半天。现在 15 分钟出初稿，省下来的时间用于采访、打磨细节、和选题对齐。

## 场景二：3 分钟完成全文校对

### 什么时候会用到

- 文章写完了，发布前需要最终检查
- 担心有错别字、语病但自己读了三遍已经读不出来了
- 涉及数据和事实引用，需要做准确性检查

### 怎么做

把全文丢给 Agent：

> "帮我校对这篇 3000 字的产品发布稿"

Agent 3 分钟返回一份校对报告：

- **错别字**："产品以经上线"→"产品已经上线"
- **语病**："这不仅提升了效率，而且还能降低了成本"→ 时态不一致
- **事实核查**：文中提到的市场规模数据建议核实最新来源
- **格式问题**：数字格式不统一、段落间缺少过渡句

> "按你的建议全部改掉"

Agent 直接输出修改后的全文。

### 效果

自己反复读三遍也可能漏错别字——人脑天然会"脑补"正确的字。Agent 不会，它逐字检查，错别字、语病、事实引用一遍过。

## 场景三：一稿改多平台，5 分钟搞定

### 什么时候会用到

- 一篇文章要同时发公众号、知乎、小红书、抖音
- 每个平台风格不一样，手动改要改四遍
- 老板说"这篇文章全渠道分发"

### 怎么做

> "这篇 3000 字的公众号文章，帮我改成知乎版、小红书版和抖音短视频脚本"

Agent 5 分钟交付三个版本，风格精准适配：

**知乎版（1500 字）：** 增加深度分析和数据引用，"先说结论"开头，结尾引导讨论

**小红书版（500 字）：** 标题加 emoji，分点罗列，口语化，结尾加互动，自动推荐话题标签

**抖音脚本（60 秒）：** 开头 3 秒抓眼球，正文 3 个要点每点 15 秒，结尾引导关注

每个版本都懂对应平台的"潜规则"——知乎用户喜欢干货和数据，小红书用户喜欢轻松和实用，抖音要前 3 秒留人。

### 效果

以前一篇文章改 4 个平台版本要大半天。现在 5 分钟出齐，各平台风格精准适配。你只需要检查一遍确认调性没问题。

## 给编辑的小建议

1. **初稿是起点，不是终点**：Agent 出的初稿结构完整但可能缺少你的个人风格和独到观点。最好的工作方式是：Agent 搭框架 → 你填灵魂。

2. **选题灵感用 Heartbeat 自动化**：在 Cron Job 里打开 Heartbeat，HEARTBEAT.md 写上"搜索行业热点和热搜话题，整理出 5 个选题方向"，间隔设为每天早上。你到公司打开一看，今天写什么心里就有数了。如果只需要工作日更新，用 Scheduled Tasks 的 Weekly 类型更精确。

3. **稿件审核推到 IM 群**：在 Agent Config → Channel 里接入飞书或钉钉。定稿后让 Agent 把文章摘要和链接推到审核群，主编直接在群里回复意见，不用来回切换工具。

4. **多平台版本一起出**：写完一篇文章后，养成习惯让 Agent 一次性出所有平台版本。5 分钟的事，能让你的内容覆盖翻倍。

5. **用 Basic Info 固定你的文风标准**：在 Soul 里写上"你是一个资深内容编辑，文风简洁有力，擅长用短句和比喻，禁止使用'赋能'、'抓手'等空话"。在 User 里注明目标受众和平台偏好——"主要产出公众号和知乎内容，面向中小企业主"。从此 Agent 的初稿风格就自动对齐你的标准。
